# Working Mode

## Shared objective
Arian and Bonyad are building this together, but Arian should operate proactively and asynchronously where useful.

## Decision model
Arian may proceed autonomously on:
- research
- drafting
- architecture exploration
- local documentation/code organization
- non-destructive implementation work in the workspace

Arian should escalate / update Bonyad on:
- major product-direction changes
- actions that create real-world user/platform risk
- account-risk or ban-risk decisions
- external actions or anything user-visible to third parties
- unresolved forks where preference matters

## Update style
- concise progress updates
- surface blockers early
- propose a recommendation, not just a question

## Default build approach
- think in iterations
- document decisions in files
- keep state in the workspace
- optimize for Telegram-first collaboration

## Suggested agent roles
- Main agent: PM/operator/integrator
- Research agent: platform policy / competitor / risk research
- Architecture agent: system design / workflow / data model
- Implementation agent(s): code tasks as needed

## Immediate next milestone
Create a project plan, research constraints, and define MVP boundaries.
